Abstract Classes in C++ Quiz

Question 1: What makes a class abstract in C++?




Explanation: In C++, a class becomes abstract when it has at least one pure virtual function (declared with = 0). There is no 'abstract' or 'interface' keyword in C++.

Question 2: Which of the following correctly declares a pure virtual function?




Explanation: The correct syntax for declaring a pure virtual function in C++ is using the '= 0' notation after the function declaration.

Question 3: What happens if you try to instantiate an abstract class?




Explanation: Trying to instantiate an abstract class directly will result in a compilation error. Abstract classes are meant to be base classes that cannot be instantiated directly.

Question 4: Complete the code to make the Animal class abstract:

class Animal {
public:
    Animal();
    virtual ~Animal();
    virtual void makeSound() const ;
};
Explanation: To make a function pure virtual and thus make the class abstract, you add '= 0' after the function declaration.

Question 5: Which of the following is true about abstract classes in C++?




Explanation: Abstract classes in C++ can have constructors, data members, and implemented methods. The only restriction is that they cannot be instantiated directly due to having at least one pure virtual function.

Question 6: What is the main purpose of an abstract class?




Explanation: The main purpose of an abstract class is to define a common interface that derived classes must implement. It provides a way to achieve polymorphism.

Question 7: In our Animal example, why did we make the makeSound() method pure virtual?




Explanation: We made the makeSound() method pure virtual because creating a generic Animal object doesn't make sense in our context. A generic "Animal" doesn't have a specific sound, only specific animals like dogs or cats do.
